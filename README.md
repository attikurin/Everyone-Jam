## 📖 プロジェクトの目的

サービス終了してしまったGoogle Jamboardを惜しむ声は、今も学校現場に多く残っています。
「複雑な高機能は要らない、シンプルで直感的に使えるあの感覚が欲しい」
「小学生にFigJamは少し難しい」
そんな先生方の声に応えるために作られた、**温かみのある日本語UI**のホワイトボードアプリです。

### 🆕 PDF / 画像インポート（v1.8.0 / Phase 8）

#### 📄 既存教材を背景として取り込み、その上に書き込み
左ツールバーの **「📥 取り込み」ボタン** または モバイルメニューから、**PDF・PNG・JPEG・GIF・WebP** ファイルを開いて現在のページ背景に貼り付けられます。学校で配られたプリントや既存の指導案を、そのまま電子黒板の上に乗せて児童と一緒に書き込めるようになります。

| 入力形式 | 動作 |
|---|---|
| **PDF**（複数ページ可） | 1ページ目は現在のページ背景に、2ページ目以降は新規ページを自動追加して各背景に配置（最大10ページ） |
| **画像**（PNG/JPEG/GIF/WebP） | クリック or ドラッグ&ドロップで現在のページの背景に配置 |

#### 🎚 フィット制御・不透明度調整・削除
- **フィットモード3種**：
  - **contain（収める）**：縦横比を保ったままボード内に収まるよう拡縮（デフォルト）
  - **cover（埋める）**：縦横比を保ったまま全面を埋める（はみ出し部分は切り取り）
  - **stretch（伸ばす）**：ボード全体を埋めるように縦横別々に拡縮
- **不透明度スライダー（10〜100%）**：背景画像を薄く表示して、上に書く文字を読みやすく
- **🗑 削除ボタン**：背景画像のみクリア（描画オブジェクトは残る）
- **初期不透明度プリセット**（取り込み時に選択）：100% / 80% / 60% / 40%

#### 🛠 技術ディテール
- **PDF.js 3.11**（CDN）で完全クライアントサイドPDFレンダリング（APIキー不要）
- **Canvas API**で画像を **最大1920px** にダウンスケール + **JPEG quality 0.82** に再エンコード → localStorage サイズを抑制
- **DataURL** として `page.backgroundImage = { src, opacity, fit }` に保存 → 再起動後も復元
- **Konva.Image** ノードとして bg-group に描画 → ペン・付箋・テキストはすべて画像の上に重なる
- **サムネイル同期**：SVG `<image>` で `preserveAspectRatio` を切り替えてフィットモードを忠実に再現
- **PNG/PDF書き出し統合**：書き出し時は背景画像も含めて1枚絵に焼き付け
- **P2P同期対応**：他端末にも背景画像がリアルタイム反映（DataURLが転送される）
- **ドラッグ&ドロップUX**：モーダル内のドロップゾーンへ直接ファイルをドロップ可能

### 🆕 投票・挙手機能（v1.10.0 / Phase 10）

#### 🗳️ クラス全員の意見をリアルタイム集計、授業参加型の議論をさらに加速
GIGA端末で一人一台の時代、**「賛成の人、手を上げて」** をデジタル化した機能です。左ツールバーの **「🗳️投票ボタン」** から質問と選択肢を作ると、ボード上に投票カードが配置され、児童はタップだけで投票 → 棒グラフがリアルタイムで伸びる — **P2P同期で全員の画面に反映**されます。

| 機能 | 内容 |
|---|---|
| **2モード切替** | 🗳️ 投票モード（最大6選択肢）/ ✋ 挙手モード（「はい！/まだです」の即答） |
| **選択肢カスタマイズ** | 質問（100文字）＋選択肢 ＋ 絵文字（10種類から選択）＋6色のカラー |
| **投票設定3軸** | 複数選択 ON/OFF / 匿名投票 ON/OFF / 結果公開タイミング3種 |
| **結果公開タイミング** | `always`（いつでも表示） / `after-vote`（自分が投票したら） / `after-close`（締め切り後） |
| **リアルタイム棒グラフ** | 投票と同時にKonvaのグループ描画が更新される（チラチラしない全面再構築方式） |
| **締め切り / 集計リセット** | 先生が達成したら「締め切る」、やり直したい場合は「リセット」 |
| **1人1票保障** | localStorage ベースの匿名 voter ID（`v_xxxxxxxx`）で同一端末からの重複投票を防止 |
| **公開タイミング同期** | リモート票は `poll-vote` 軸量メッセージでだけブロードキャスト（全体状態同期不要） |

#### 🛠 技術ディテール
- **漂うエモジやカラー**で孩も記号に迫られず参加しやすいUIだし、リサイズコーナーでボードスケールに合わせて拡大（8～15段階可視性）
- **`addObjectToPage(obj)` を使う一育化フロー** — historyプッシュ、ストレージ保存、P2Pブロードキャスト、サムネイル更新をすべて代行
- **サイズ変更対応** — Konva.Transformer でカードを広げると、`transformend` で `obj.width` を入れ替えてグループを再構築（テキストサイズやレイアウトが坊れない）
- **投票データ構造** — `votes: { [voterId]: optionId | optionId[] }`。単一選択は文字列、複数選択は配列。同一選択肢を再タップで取消し可能
- **P2P同期** — 投票イベントを `poll-vote` メッセージでピンポイント送信し、受信側は `applyRemoteVote()` で該当カードだけ部分再描画
- **PNG/PDF出力対応** — `export.js` の `createNodeFromDataNoHandler` でも `case 'poll'` を `createPollNode` に委譲、他オブジェクトと同じように1枚絵に焼き付ける
- **PWAオフライン対応** — sw.js に `poll.js?v=1.10.0` をプリキャッシュ追加

### 🆕 手書き文字認識 OCR（v1.9.0 / Phase 9）

#### 🔍 児童の手書き答案をテキスト化 → 付箋やテキストとしてボードに挿入
左ツールバーの **「🔍 OCRボタン」** を押すと OCRモードが起動し、ボード上で **ドラッグして範囲を選択するだけ** でそのエリアの手書き文字を認識、テキストオブジェクトや付箋としてボードに挿入できます。タブレットで受け取った児童の答案を、その場で **クラスチャートに並べて議論させる** という使い方に最適です。

| 機能 | 内容 |
|---|---|
| **ドラッグ選択** | OCRモード中はボード上で範囲をドラッグ → 破線枠でプレビュー → 離すと自動でOCR開始 |
| **ファイルアップロード** | スマホで撮った答案画像を直接アップロードして認識（最大20MB / PNG/JPEG/WebP） |
| **リアルタイム進捗** | エンジン初期化・言語データ読み込み・認識中の進捗バーと日本語ラベル |
| **言語選択** | 日本語+英語（推奨）/ 日本語のみ / 英語のみ — 途中で切り替え可 |
| **結果編集** | 認識結果はその場で textarea で修正可能（誤認識を信頼しすぎないUX） |
| **挿入モード5種** | テキストオブジェクトとして / 6色の付箋として / クリップボードコピー |
| **再認識** | 同じ画像を言語を変えてやり直し可能 |

#### 🛠 技術ディテール
- **Tesseract.js v5**（CDN） — 完全クライアントサイドOCR、APIキー不要、外部送信一切なし
- **Workerシングルトンパターン** — 一度作ったワーカーを使い回し、言語データ（jpn約 13MB）の再ダウンロードを回避
- **800px以上にアップスケール** — 選択エリアをCanvasで拡大し、手書き文字の認識精度を向上
- **`renderPageToDataURLIndex` を再利用** — Phase 8のエクスポート描画を流用して背景画像+オブジェクトごとキャプチャ
- **PWAオフライン対応** — sw.js に Tesseract もキャッシュ追加（初回オンライン以降はオフラインでもOCR可）
- **ドラッグ選択とツールの互換性** — OCRモード中はステージイベントを优先受け取り、他ツールの誤作動を防止

### 🆕 PWA化（v1.6.0 / Phase 6）

#### 📲 ホーム画面に追加（インストール可能アプリ）
- **Web App Manifest**（`manifest.json`）でインストール可能化
- **Android / Chrome**：`beforeinstallprompt` を検出して **ヘッダーの「📲 インストール」ボタン** を表示。ワンタップでホーム画面追加ダイアログを起動
- **iOS Safari**：プラットフォーム自動判定後に **手順ガイドモーダル**（共有→ホーム画面に追加）を表示
- **Desktop（Edge/Chrome）**：アドレスバーのインストールアイコン + アプリ内ボタンの両方から導線
- **standaloneモード**：起動時にアドレスバー無しの全画面アプリとして開く（`display: standalone`）
- **アプリショートカット**：ホーム画面アイコン長押しから「新しいボード」「ボード一覧」へ直行（Android対応）
- **iOSセーフエリア対応**：standalone時もノッチ・ホームバーを回避する余白を確保

#### 🌐 オフライン動作（Service Worker）
- **3層キャッシュ戦略**：
  - **STATIC（cacheFirst）**：HTML/CSS/JS/画像/アイコンを事前キャッシュ → 一度開いたら通信不要
  - **RUNTIME（networkFirst）**：HTMLは最新を優先、失敗時にキャッシュへフォールバック
  - **CDN（staleWhileRevalidate）**：Konva/Tailwind/Font Awesome等のCDNは即時キャッシュ返却＋裏で更新
- **完全オフライン時のフォールバックHTML**：通信が完全に失えても「オフライン中」を伝える静的ページを表示
- **オフラインバッジ**：オフラインになるとヘッダーに **「🌐 オフライン」黄色バッジ** が点滅表示（`navigator.onLine` + online/offlineイベント）
- **オンライン復帰時自動消失**：再接続を検出すると緑色トーストを表示してバッジを非表示

#### 🔄 自動アップデート検知
- 新バージョンのService Workerが見つかると **画面下部にスライドインバナー** が出現
- 「**今すぐ更新**」ボタンで `skipWaiting` + `controllerchange` → 自動リロード
- 「あとで」ボタンでバナーを閉じて手動更新も可能
- キャッシュバージョン制（`v1.6.0`）で古いキャッシュを安全に廃棄

- maskableアイコン（192×192 / 512×512）：Androidのセーフゾーンに収まるフルブリードデザイン
- apple-touch-icon（180×180）：iOSホーム画面用
- favicon（32×32 / 16×16）：ブラウザタブ用

### BroadcastChannel メッセージ
チャンネル名: `minnanojam_<boardId>`
```js
{ type: 'board-update', source: 'client_xxx', data: { title, pages } }
{ type: 'request-sync', source: 'client_xxx' }
```

### P2P メッセージタイプ（v1.4.0 拡張後）
| type | 意味 | 主なペイロード |
|---|---|---|
| `snapshot` | ホストからのボード全体状態 | { boardTitle, pages } |
| `state-update` | ボード全体の差分同期 | { boardTitle, pages } |
| `cursor` | カーソル位置 | { x, y, color, name, page } |
| `presence` | 参加者情報 | { name, color } |
| `peer-joined` / `peer-left` | 参入退出 | { peerId } |
| **`selection`** 🆕 | 選択オブジェクト | { ids[], color, name, page } |
| **`comment-update`** 🆕 | コメント追加・修正・返信・解決 | { action, pageIndex, comment } |

拡張ハンドラは `window.p2pRegisterHandler(type, handler)` で外部モジュールから追加可能。

## 🛠 共同編集の仕組み（技術的詳細）

```
[先生 A] ── PeerJS シグナリング ──┐
   │ (open)                      │
   ▼                              │
[ホスト] ◀── DataConnection ──── [生徒 B]
   ▲                              │
   └─── DataConnection ────── [生徒 C]
        ↑（ホストが中継）
```

- **ホストID** は `mnj-host-<boardId>` として決定論的に生成
  - 最初の人が `Peer(hostId)` の取得に成功 → ホスト
  - 既に取られていれば → クライアントとしてホストへ接続
- **メッセージ種別**：`hello` / `request-snapshot` / `snapshot` / `state-update` / `cursor` / `presence` / `peer-joined` / `peer-left`
- **state-update** はボード全文（pages配列）を送信し、80msのスロットリング付き
- **cursor** は20fps（50ms）スロットリング、ボード座標で送信
- **STUNサーバー**：Google `stun:stun.l.google.com:19302` / Twilio
- **シグナリング**：PeerJS無料公開サーバー（0.peerjs.com）
